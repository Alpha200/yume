== 6.1 Scenario: User Sends a Message via Matrix

This is the primary interaction flow.

[mermaid]
....
sequenceDiagram
    actor U as User (Matrix)
    participant M as MatrixClientService
    participant R as RequestRouterService
    participant CS as ConversationSummarizerAgent
    participant MS as MemorySummarizerService
    participant RA as RequestRouterAgent
    participant RP as ResourceProviderService
    participant A as Specialist Agent
    participant MM as MemoryManagerExecutorService

    U->>M: sends message in room
    M->>R: UserMessageEvent
    R->>CS: summarise(conversationHistory)
    CS-->>R: summary
    R->>MS: getMemorySummaries()
    MS-->>R: summaries (preferences, observations, reminders)
    R->>RA: route(message, summaries)
    RA-->>R: YumeAgentType + required resources
    R->>RP: fetchResources(requiredResources)
    RP-->>R: context string
    R->>A: chat(message, memories, resources)
    A-->>R: response
    R-->>MM: updateMemory (async, non-blocking)
    R->>M: sendReply(response)
    M->>U: reply in Matrix room
....

'''

== 6.2 Scenario: Proactive Scheduler Run

The AI determines its own next execution time.

[mermaid]
....
sequenceDiagram
    participant SS as SchedulerService
    participant SE as ScheduleExecutorService
    participant R as RequestRouterService
    participant SA as SchedulerAgent
    participant DB as MongoDB

    SS->>SE: fires at AI-decided time
    SE->>R: runFromScheduler()
    Note over R: same routing flow as 6.1 with scheduler context
    R-->>SE: response
    SE->>DB: persist SchedulerRun (status, topic, duration)
    SE->>SS: SchedulerExecutedEvent
    SS->>SA: decideNextRun(context)
    SA-->>SS: nextRun timestamp + topic
    SS->>SS: schedule next task
....

'''

== 6.8 Scenario: Garmin Health Snapshot

[mermaid]
....
sequenceDiagram
    participant RP as ResourceProviderService
    participant GCS as GarminConnectService
    participant GDF as GarminConnectDataFetcherService
    participant Cache as Caffeine Cache
    participant MCP as Garmin MCP Server

    RP->>GCS: getHealthSnapshot()
    GCS->>GDF: fetchSnapshot()
    GDF->>Cache: check cache
    alt cache hit
        Cache-->>GDF: cached GarminHealthStatus
    else cache miss
        GDF->>MCP: SSE connect + tools/call snapshot
        MCP-->>GDF: health data (sleep, HRV, steps, training load)
        GDF->>Cache: store result
    end
    GDF-->>GCS: GarminHealthStatus
    GCS-->>RP: formatted health string
....
