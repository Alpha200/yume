== 5.4 Level 2 — Service Subsystem

=== Orchestration Services

[cols="2,4", options="header"]
|===
| Service | Responsibility

| `RequestRouterService`
| Central dispatcher. Listens for `UserMessageEvent`, `UserReactionEvent`. Runs summariser, loads memories, invokes `RequestRouterAgent`, dispatches to specialist. Also handles geofence events and scheduler-triggered runs.

| `ResourceProviderService`
| Aggregates context strings from all data sources (weather, day plan, calendar, location, memories, health, Strava, scheduler logs). Called by the router before specialist invocation.

| `SchedulerService`
| Manages a `ThreadPoolTaskScheduler`. Listens for `DayPlanUpdatedEvent` and `SchedulerExecutedEvent` to re-schedule.

| `ScheduleExecutorService`
| Executes a single scheduled run: calls router, persists log, publishes `SchedulerExecutedEvent`.
|===

=== Memory Services

[cols="2,4", options="header"]
|===
| Service | Responsibility

| `MemoryManagerService`
| CRUD for `MemoryEntry` subtypes in MongoDB + pgvector.

| `MemoryManagerExecutorService`
| Async wrapper for `MemoryManagerAgent`; triggers re-summarisation; contains scheduled janitor.

| `MemorySummarizerService`
| Maintains per-`MemoryType` summaries as in-memory cache; rebuilt on startup and after memory changes.
|===

=== Matrix Service

[cols="2,4", options="header"]
|===
| Service | Responsibility

| `MatrixClientService`
| Trixnity-based Matrix bot. Subscribes to room messages and reactions on startup. Publishes Spring events. Sends replies.
|===

=== Day Plan Services

[cols="2,4", options="header"]
|===
| Service | Responsibility

| `DayPlanService`
| CRUD for `DayPlan` MongoDB documents.

| `DayPlanExecutorService`
| Scheduled (2:05 AM cron) + ad-hoc day plan generation using `DayPlanAgent`.
|===

=== External Integration Services

[cols="2,3", options="header"]
|===
| Service | External System

| `CalendarService` | CalDAV / Nextcloud (caldav4j)
| `WeatherService` | OpenWeatherMap One Call API
| `EfaService` | EFA German public transit XML API
| `KitchenOwlService` | KitchenOwl REST API
| `StravaActivityService` | Strava REST API
| `StravaAuthService` | Strava OAuth2
| `StravaWebhookService` | Strava Webhook subscription management
| `GarminConnectService` | Garmin Connect (via MCP server)
| `GarminConnectDataFetcherService` | MCP over SSE using MCP Kotlin SDK
|===

=== Location Services

[cols="2,4", options="header"]
|===
| Service | Responsibility

| `LocationRetrieverService`
| Reads current user location from Home Assistant `person` entity. Cached.

| `LocationService`
| Formats location as geofence name + place name + distance to home.

| `ReverseGeocodingService`
| Nominatim API calls.

| `GeofenceEventLogService`
| Records geofence ENTER/LEAVE events.
|===

=== Conversation Services

[cols="2,4", options="header"]
|===
| Service | Responsibility

| `ConversationHistoryManagerService`
| Persists and retrieves recent Matrix conversation history from MongoDB.

| `InteractionTrackerService`
| Implements `ChatModelListener`; records all LLM requests/responses in an in-memory `ArrayDeque`.
|===

== 5.6 Level 2 — Data Model

=== MongoDB Collections

[cols="2,2,4", options="header"]
|===
| Collection | Document Type | Key Fields

| `memories`
| `MemoryDocument` (sealed: `UserPreferenceEntry`, `UserObservationEntry`, `ReminderEntry`)
| `id`, `content`, `type`, `place`, `createdAt`, `modifiedAt`, type-specific fields

| `day_plans`
| `DayPlan`
| `id`, `date`, `items` (list of `DayPlanItem`), `summary`, `calendarEventHashes`

| `scheduler_runs`
| `SchedulerRun`
| `scheduledTime`, `actualExecutionTime`, `topic`, `reason`, `status`, `aiResponse`, `executionDurationMs`

| `conversation_history`
| `ConversationHistoryEntry`
| `content`, `type` (USER/ASSISTANT), `timestamp`, `eventId`

| `geofence_event_log`
| `GeofenceEventLog`
| `geofenceName`, `eventType` (ENTER/LEAVE), `timestamp`

| `strava_credentials`
| `StravaCredentials`
| `accessToken`, `refreshToken`, `expiresAt`
|===

=== PostgreSQL Tables

[cols="2,4", options="header"]
|===
| Table | Purpose

| `yume_embeddings`
| pgvector embeddings for memory entries; used for semantic search by `PgVectorEmbeddingStore`
|===
